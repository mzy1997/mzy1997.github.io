JSR223-Assertion

发表于2026-03-14|更新于2026-03-14
|浏览量:

最近在为JMeter的接口做断言，一开始想选用JMeter提供的响应断言或Json断言，但最后发现这俩玩意儿不太适合我，因为我已经习惯了使用chai.js一套撸完所有断言的方式，而JMeter提供的两种断言限制太多，有时候需要建多个断言才能校验完所有字段，最后我选择了JSR223 Assertion.

The JSR223 Assertion allows JSR223 script code to be used to check the status of the previous sample


因为groovy在JMeter中解析性能较好，上手也快，所以选择用groovy来写校验脚本
Documentation：

https://jmeter.apache.org/usermanual/component_reference.html#JSR223_Assertion

http://www.groovy-lang.org/documentation.html

https://www.w3cschool.cn/groovy/


技术选型虽然做好了，但是存在一个技术难点，其他语言的assert库都是校验不通过自动抛出异常、输出异常信息，那么在JMeter中如何实现这一点呢？

遇事不决先查官网，果然官网就是yyds

The script can check various aspects of the SampleResult. If an error is detected, the script should use AssertionResult.setFailureMessage(“message”) and AssertionResult.setFailure(true).

官网提到了AssertionResult类，以及该类的两个与异常相关的方法，于是我就去查了下这个类。

setFailure：传入一个布尔值，标志该assertion是否失败

setFailureMessage：传入一个字符串，作为异常信息

因此，大概思路已确定下来，我们可以针对status code、response body等内容进行校验，如果不通过，就将异常信息在failmessage变量里拼接，再调用setFailMessage方法，最后待所有校验做完了，我们再判断一下failmessage字符串是否为空，为空则表示校验通过，不为空我们就调用setFailure，并传入true标记该校验已失败。
