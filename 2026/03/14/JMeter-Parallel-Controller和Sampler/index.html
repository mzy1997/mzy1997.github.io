JMeter-Parallel-Controller和Sampler

发表于2026-03-14|更新于2026-03-14
|浏览量:

前言

Parallel Controller & Sampler是由BlazeMeter开发的一款JMeter插件，用来解决并行调用接口的场景，该款插件主要分为一个Controller（控制器）、一个Sampler（取样器）。

How to install

进入JMeter Plugins Manager，搜索Parallel Controller & Sampler

Parallel Controller

Parallel Controller makes all of its children to run in parallel to each other

  • 说明：引用Parallel Controller在JMeter plugins Manager里的描述，其作用就是让并行控制器下的各个子节点并行运作。

  • 使用背景：接口自动化工作中，常遇到一个接口入参了多个业务的主键ID，有时候需要去各个表内筛选出我们需要的ID，但当表内数据较多且筛选字段没有建索引时，查询时间就比较长，再加上多个查询请求之间线性调用，接口执行时间也是线性增长。这个时候就可以使用Parallel Controller，将彼此之间没有依赖的查询请求作为该控制器的子节点进行并行调用。

  • Documentation:

    • https://github.com/Blazemeter/jmeter-bzm-plugins/blob/master/parallel/Parallel.md
    • https://www.blazemeter.com/blog/how-to-use-the-parallel-controller-in-jmeter

  • 示例：

    创建一个Parallel控制器，再在该控制器下创建两个简单控制器，简单控制器下的取样器都会并行运作。

    运行效果：

    可以看到Parallel Controller下的http取样器并不是按照从上到下的顺序执行的，两个简单控制器是并行调用的，这样在准备数据时可以执行不少的时间。

Parallel Sampler

Parallel Sampler allows make requests like embedded resources for URLs from list in GUI, without making request for main page.

  • 说明：该取样器通常用于并行调用获取内部资源的接口，从而达到优化接口结构的效果。

  • 使用方法：

    只需要在Parallel Sampler中输入所有需要并行接口的URL即可
