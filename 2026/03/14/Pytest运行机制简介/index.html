1. Pytest运行机制简介

  • 如果有指定目录，就从指定的目录查找；没有指定，默认当前目录
  • 查找该目录及其子目录下的测试文件
  • 测试文件名称需满足 test_*.py 或 *_test.py 的格式
  • 文件内的函数名称需以 test 开头
  • 文件内的类名称需以 Test 开头，并会筛选掉有 init 函数的的类
  • 再查找类中以 Test_ 开头的类方法

2.应当遵守的命名规范

  为了保证我们编写的用例能按照预想的方式运行，我们需要遵守以下命名规范：

  • 测试文件应遵守 test_filename.py 或 filename_test.py 的命名方式；
  • 测试函数应遵守 test_casename 的命名方式;
  • 测试类应该遵守 TestClassName 的命名方式;

3. 修改默认的运行规则

 修改 pytest.ini 文件，该文件位置防止与项目根目录。

  • 修改默认的测试文件命名方式

    如：测试文件前缀需要统一为项目名，purchase_*.py

  • 修改默认的测试函数命名方式

    如：测试函数需要以case为前缀，case_*

  • 修改默认的测试类命名方式

    如：测试类需要以TestClass为前缀，TestClass*
